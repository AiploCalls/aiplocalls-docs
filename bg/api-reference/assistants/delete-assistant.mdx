---
---
title: "Изтриване на асистент"
api: "DELETE /user/assistant/{id}"
description: "Изтриване на конкретен AI асистент"
---

Този endpoint ви позволява да изтриете окончателно AI асистент, който принадлежи на автентикирания потребител.

### Path Параметри

<ParamField path="id" type="integer" required>
  Уникалният идентификатор на асистента за изтриване
</ParamField>

### Отговор

<ResponseField name="message" type="string">
  Съобщение за потвърждение, указващо че асистентът е изтрит успешно
</ResponseField>

### Отговори при грешка

<ResponseField name="404 Not Found">
  <Expandable title="Отговор при грешка">
    <ResponseField name="message" type="string">
      Съобщение за грешка, когато асистентът не е намерен или не принадлежи на автентикирания потребител
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="500 Internal Server Error">
  <Expandable title="Отговор при грешка">
    <ResponseField name="message" type="string">
      Съобщение за грешка, указващо неуспешно изтриване на асистента
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>

```json 200 Success Response
{
  "message": "Assistant deleted successfully"
}
```

```json 404 Not Found
{
  "message": "Assistant not found"
}
```

```json 500 Internal Server Error
{
  "message": "Assistant deletion failed"
}
```

</ResponseExample>

### Бележки

- Само асистенти, които принадлежат на автентикирания потребител, могат да бъдат изтрити
- След като асистент бъде изтрит, той не може да бъде възстановен
- Това действие ще премахне окончателно асистента и цялата му конфигурация
- Ако асистентът в момента обработва активни повиквания, тези повиквания могат да бъдат засегнати
