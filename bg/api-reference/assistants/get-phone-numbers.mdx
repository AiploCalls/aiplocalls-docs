api: "GET /user/assistants/phone-numbers"
description: "Извличане на всички налични телефонни номера за възлагане на асистенти"
---

Тази крайна точка връща списък с телефонни номера, притежавани от удостоверения потребител, които могат да бъдат възложени на асистенти, с опционално филтриране по тип асистент.

### Query Parameters

<ParamField query="type" type="string" optional>
  Филтриране на телефонни номера по тип асистент. Опции: `inbound`, `outbound`
</ParamField>

### Response fields

<ResponseField name="data" type="array">
  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      Уникалният идентификатор на телефонния номер
    </ResponseField>

    <ResponseField name="phone_number" type="string">
      Телефонният номер в E.164 формат
    </ResponseField>

    <ResponseField name="country_code" type="string">
      Кодът на страната за телефонния номер
    </ResponseField>

    <ResponseField name="type_label" type="string">
      Етикетът на типа за четене от човек за телефонния номер
    </ResponseField>

    <ResponseField name="is_available" type="boolean">
      Дали телефонният номер е наличен за възлагане (в момента не е възложен на входящ асистент)
    </ResponseField>
  </Expandable>
</ResponseField>

### Бележки

- Връщат се само телефонни номера, притежавани от удостоверения потребител
- Когато е зададен `type=inbound`, номерата за caller ID и вече възложените номера се филтрират
- Полето `is_available` показва дали номерът може да бъде възложен на нов входящ асистент
- Телефонните номера се връщат във формат на колекция от стойности (индексиран масив)
- Използвайте полето `id` при възлагане на телефонни номера на асистенти
