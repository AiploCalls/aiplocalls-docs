Този endpoint връща списък с WhatsApp Business изпращачи (телефонни номера), конфигурирани за вашия акаунт. Използвайте това за извличане на ID-та на изпращачи, необходими за изпращане на съобщения.

### Query Параметри

<ParamField query="status" type="string" optional>
  Филтриране на изпращачи по статус. По подразбиране: `online`. Използвайте `all` за да върнете всички изпращачи независимо от статуса.
</ParamField>

### Response Полета

<ResponseField name="data" type="array">
  <Expandable title="Свойства на обекта изпращач">
    <ResponseField name="id" type="integer">
      Уникалният идентификатор на WhatsApp изпращача. Използвайте този ID при изпращане на съобщения.
    </ResponseField>

    <ResponseField name="phone_number" type="string">
      Телефонният номер на изпращача в E.164 формат (напр., `+14155551234`)
    </ResponseField>

    <ResponseField name="display_name" type="string">
      Показваното име на WhatsApp Business (името на бизнеса, показвано на получателите)
    </ResponseField>

    <ResponseField name="status" type="string">
      Текущият статус на изпращача: `online` или `offline`
    </ResponseField>

    <ResponseField name="quality_rating" type="string">
      Рейтингът на качеството от Meta за този изпращач: `GREEN`, `YELLOW`, или `RED`
    </ResponseField>

    <ResponseField name="messaging_limit" type="integer">
      Текущата граница за съобщения (брой уникални получатели за 24 часа). `null` означава неограничено.
    </ResponseField>

    <ResponseField name="messaging_limit_formatted" type="string">
      Четима за човека граница за съобщения (напр., `1,000 / 24hr`, `10,000 / 24hr`, или `Unlimited`)
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>

```bash cURL
curl -X GET "https://call.aiployees.com/api/user/whatsapp/senders" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```bash Филтрирай всички изпращачи
curl -X GET "https://call.aiployees.com/api/user/whatsapp/senders?status=all" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```javascript JavaScript
const response = await fetch(
  'https://call.aiployees.com/api/user/whatsapp/senders',
  {
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY'
    }
  }
);

const data = await response.json();
console.log(data.data); // Array of senders
```

```python Python
import requests

response = requests.get(
    'https://call.aiployees.com/api/user/whatsapp/senders',
    headers={'Authorization': 'Bearer YOUR_API_KEY'}
)

senders = response.json()['data']
for sender in senders:
    print(f"{sender['display_name']}: {sender['phone_number']}")
```

### Забележки

- По подразбиране се връщат само изпращачи със статус `online`. Използвайте `?status=all` за да включите офлайн изпращачи.
- Статусът на изпращача се синхронизира с Meta на всеки 5 минути автоматично.
- Полето `id` е това, което трябва да подадете като `sender_id` при изпращане на съобщения.
- Рейтингът на качеството влияе върху вашите лимити за съобщения. Рейтинг `RED` може да ограничи способността ви да изпращате съобщения.
