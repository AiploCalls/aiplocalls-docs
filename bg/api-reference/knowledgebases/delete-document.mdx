---
title: "Изтриване на документ"
api: "DELETE /user/knowledgebases/{knowledgebaseId}/documents/{documentId}"
description: "Изтриване на документ от база от знания"
---

Този endpoint окончателно изтрива документ от база от знания. Векторните вграждания на документа също се премахват от векторната база данни.

### Path Parameters

<ParamField path="knowledgebaseId" type="integer" required>
  Уникалният идентификатор на базата от знания
</ParamField>

<ParamField path="documentId" type="integer" required>
  Уникалният идентификатор на документа за изтриване
</ParamField>

### Response

<ResponseField name="message" type="string">
  Съобщение за успех
</ResponseField>

<ResponseExample>

```json 200 Response
{
  "message": "Document deleted successfully."
}
```

```json 404 Knowledgebase Not Found
{
  "error": "Knowledgebase not found."
}
```

```json 404 Document Not Found
{
  "error": "Document not found."
}
```

```json 500 Error
{
  "error": "Failed to delete document. Please try again."
}
```

</ResponseExample>

### Какво се изтрива

Когато изтривате документ:

- **Записът на документа** се премахва от базата данни
- **Векторните вграждания** се изтриват от векторната база данни
- **Обработеното съдържание** се премахва от външните хранилища за документи
- **Статусът на родителската база от знания** се актуализира автоматично

<Warning>
Това действие е необратимо. Документът и цялото му обработено съдържание ще бъдат окончателно изтрити.
</Warning>

<Note>
Ако това беше последният документ в базата от знания, статусът на базата от знания ще се промени на `empty`.
</Note>
