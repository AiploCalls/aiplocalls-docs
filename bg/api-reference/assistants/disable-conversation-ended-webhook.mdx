---
title: "Деактивиране на webhook за приключена конversация"
api: "POST /user/assistants/disable-conversation-ended-webhook"
description: "Деактивиране на webhook уведомления за приключена конversация за асистент"
---

Този endpoint деактивира webhook уведомленията за приключена конversация за асистент, спирайки доставката на актуализации когато чат конversациите приключат.

### Request Body

<ParamField body="assistant_id" type="integer" required>
  ID на асистента за който да се деактивира webhook за приключена конversация
</ParamField>

### Response

<ResponseField name="message" type="string">
  Съобщение за успех потвърждаващо че webhook е деактивиран
</ResponseField>

<ResponseField name="data" type="array">
  Празен масив (запазен за бъдеща употреба)
</ResponseField>

### Error Responses

<ResponseField name="404 Not Found">
  <Expandable title="Отговор при грешка">
    <ResponseField name="message" type="string">
      Съобщение за грешка когато асистентът не е намерен или не принадлежи на автентикирания потребител
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="422 Validation Error">
  <Expandable title="Отговор при грешка">
    <ResponseField name="message" type="string">
      Съобщение за грешка указващо неуспешна валидация
    </ResponseField>
    <ResponseField name="errors" type="object">
      Подробни грешки при валидация за всяко поле
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>

```json 200 Success Response
{
  "message": "Conversation ended webhook disabled successfully",
  "data": []
}
```

```json 404 Not Found
{
  "message": "Assistant not found"
}
```

```json 422 Validation Error
{
  "message": "The given data was invalid.",
  "errors": {
    "assistant_id": [
      "The assistant id field is required."
    ]
  }
}
```

</ResponseExample>
