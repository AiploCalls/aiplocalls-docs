Научете как да конфигурирате всеки тип widget с подробни обяснения на всяка опция.

## Създаване на Widget-и

1. Отидете в персонализирано табло
2. Кликнете **"Управление на Widget-и"** след това **"Създай нов Widget"**
3. Изберете тип widget
4. Следвайте wizard-а от 2-3 стъпки
5. Запазете и добавете в таблото

## Stat Widget-и

Показване на една метрика с незадължително сравнение и тенденчна диаграма.

### Стъпка 1: Основни настройки

**Име на Widget**
- Вътрешен идентификатор за списъка с widget-и
- Не се показва в таблото
- Пример: "Общо повиквания този месец"

**Таблица с данни**
- Изберете източник на данни: calls, leads, campaigns, assistants, phone_numbers, sms
- Определя наличните колони

**Агрегация**
- `COUNT` - Общ брой записи (не е нужна колона)
- `SUM` - Сума от стойностите (изисква избор на колона)
- `AVG` - Средна стойност (изисква избор на колона)
- `MAX` - Максимална стойност (изисква избор на колона)
- `MIN` - Минимална стойност (изисква избор на колона)

**Колона** (ако е избрано SUM/AVG/MAX/MIN)
- Изберете числова колона за агрегиране
- Пример: `duration` за средна продължителност на разговора
- Не е нужна за COUNT

### Стъпка 2: Показване и филтри

**Етикет** (задължителен)
- Основно заглавие, показано на widget-а
- Пример: "Общо повиквания"

**Описание**
- Незадължителен текст под стойността
- Скрит ако сравнението е активирано
- Пример: "Повиквания направени този месец"

**Цвят**
- primary (син), success (зелен), warning (жълт), danger (червен), info (циан), gray
- Изберете според типа метрика (зелен за положителни, червен за отрицателни)

**Покажи мини диаграма**
- Показва малка линейна диаграма с дневни тенденции
- Използва същия интервал от дати като статистиката

**Сравнение с предишен период**
- Показва процентна промяна спрямо предишния период
- Пример: Последни 7 дни срещу предишните 7 дни
- Показва "+15%" или "-8%" със стрелка нагоре/надолу
- Замества полето за описание

**Интервал от дати**
- Филтриране по дата на създаване
- Опции: Днес, 1, 7, 14, 30, 60, 90, 180, 365 дни

**Условия** (незадължителни)
- Добавете филтри за ограничаване на данните
- Множество условия са комбинирани с AND
- Вижте секция [Филтриране на данни](#filtering-data)

### Примери

**Общо повиквания**
```
Агрегация: COUNT
Таблица: calls
Етикет: "Общо повиквания"
Описание: "Последни 30 дни"
Цвят: primary
Интервал от дати: 30 дни
```

**Средна продължителност на разговора**
```
Агрегация: AVG
Таблица: calls
Колона: duration
Етикет: "Ср. продължителност"
Цвят: info
Покажи мини диаграма: Да
Интервал от дати: 30 дни
```

**Успешни срещи**
```
Агрегация: COUNT
Таблица: calls
Етикет: "Резервирани срещи"
Цвят: success
Сравнение: Да
Интервал от дати: 7 дни
Условия:
  - status = completed
  - evaluation.appointment_booked = true
```

## Chart Widget-и

Визуализирайте тенденции с 8 типа диаграми.

### Стъпка 1: Основни настройки

Същото като Stat widget-и, плюс:

**Тип диаграма**
- `line` - Тенденции във времето
- `bar` - Сравнение на категории или периоди от време
- `area` - Запълнена тенденчна диаграма
- `pie` - Пропорции (изисква Group By)
- `donut` - Пай с дупка в центъра (изисква Group By)
- `polarArea` - Кръгла с променлив радиус (изисква Group By)
- `radar` - Сравнение с множество оси
- `radialBar` - Единичен индикатор за напредък

### Стъпка 2: Показване и филтри

**Етикет** (задължителен)
- Заглавие на диаграмата

**Описание**
- Незадължителен подзаглавен текст под диаграмата
- Пример: "Производителност за последните 30 дни"

**Цвят**
- Основен цвят на диаграмата
- За pie/donut използва многоцветна палитра (син, зелен, оранжев, червен, лилав, розов)

**Интервал от дати**
- Същото като статистиките

**Group By** ИЛИ **Group By Period** (взаимно изключващи се)

**Group By** - Категориално групиране
- Групиране на данните по стойности от колона
- Примери: Status, Campaign Id, Assistant Name
- **Задължително** за pie/donut/polarArea диаграми
- Незадължително за bar/radar диаграми
- **Не може да се използва** с Group By Period

**Group By Period** - Групиране по време
- Групиране по времеви интервали
- Опции: hour, day, week, month
- Използва се за line/bar/area диаграми
- **Не може да се използва** с Group By

**Условия**
- Същото като статистиките

### Стъпка 3: Разширени настройки (само за диаграми)

**Покажи мрежови линии** (само line/bar/area)
- Хоризонтални/вертикални линии на фона
- Помагат за четене на точни стойности
- По подразбиране: Да

**Покажи легенда**
- Етикети с цветово кодиране
- По подразбиране: Да за pie/donut/polarArea, Не за останалите
- Позиция: Долу от диаграмата

**Покажи подсказки**
- Изплуващи прозорчета при посочване с точни стойности
- По подразбиране: Да (препоръчително)

**Покажи етикети с данни**
- Стойности показани директно на елементите на диаграмата
- По подразбиране: Да за pie/donut/polarArea, Не за останалите
- Може да претрупа line/bar диаграмите

**Покажи лента с инструменти** (само line/bar/area)
- Контроли за увеличение и изтегляне
- По подразбиране: Не

**Извивка на линията** (само line/area)
- `smooth` - Закръглени криви (по подразбиране, визуално привлекателно)
- `straight` - Директни линии (точно)
- `stepline` - Стъпаловидно (добро за дискретни събития)

**Дебелина на линията** (само line/bar/area)
- Дебелина в пиксели (1-10)
- По подразбиране: 2px

**Височина на диаграмата**
- Вертикално пространство в пиксели (200-800)
- По подразбиране: 300px
- По-високи диаграми са по-добри за повече точки с данни

**Активирай градиент** (само radialBar)
- Ефект с цветен градиент
- По подразбиране: Не

**Цвят на градиента** (ако градиентът е активиран)
- Краен цвят за градиента
- По подразбиране: Зелен (#10b981)

**Изчисли като процент** (само radialBar)
- Автоматично изчислява процент от условията
- Формула: (изпълнени условия / общо) × 100
- **Изисква:** Първо трябва да се добави поне едно условие
- Пример: Условие "status = completed" изчислява завършени лийдове / общо лийдове × 100
- Появява се след добавяне на условия в Стъпка 2

**Покажи долен колонтитул с числа** (само radialBar)
- Показва "X постигнати / Y общо" под radialBar диаграмата
- Видим само когато "Изчисли като процент" е активирано
- Показва точни числа заедно с процента
- Пример за долен колонтитул: "150 постигнати / 1,000 общо"

**Начален ъгъл** (само radialBar)
- Където започва дъгата в градуси (-360° до 360°)
- По подразбиране: -140°
- Обичайни стойности: -140° (по подразбиране), -90° (лев полукръг), 0° (отдолу)

**Краен ъгъл** (само radialBar)
- Където завършва дъгата в градуси (-360° до 360°)
- По подразбиране: 130°
- **Валидация:** Общата дъга (край - начало) не може да надвишава 360°
- Обичайни стойности: 130° (по подразбиране), 90° (десен полукръг), 180° (половин кръг), 270° (три четвърти)

**Размер на дупката** (само radialBar)
- Размер на вътрешния кръг като процент (0-100%)
- По подразбиране: 60%
- По-висока стойност = по-тънък пръстен, по-ниска стойност = по-дебел пръстен

**Dash Array** (само radialBar)
- Разстояние за пунктиран пръстеновиден модел (0-50)
- По подразбиране: 10
- 0 = непрекъсната линия, по-високи числа = повече разстояние между тиретата

### Ръководство за типове диаграми

**Линейна диаграма**
- **Използвайте за:** Дневни/седмични тенденции, непрекъснати данни
- **Изисква:** Group By Period
- **Пример:** Повиквания на ден за последните 30 дни

**Стълбовидна диаграма**
- **Използвайте за:** Сравнение на категории, периоди от време
- **Може да използва:** Group By (категории) ИЛИ Group By Period (време)
- **Пример:** Повиквания по кампания, почасово разпределение

**Area диаграма**
- **Използвайте за:** Същото като линейна, подчертава обема
- **Изисква:** Group By Period
- **Пример:** Тенденция в обема на повикванията

**Pie/Donut диаграма**
- **Използвайте за:** Пропорции, проценти
- **Изисква:** Group By (НЕ Group By Period)
- **Пример:** Разбивка на статуса на повикванията

**Polar Area**
- **Използвайте за:** Като pie, но показва величината чрез радиуса
- **Изисква:** Group By
- **Пример:** Принос на кампанията

**Radar диаграма**
- **Използвайте за:** Сравнение с множество променливи
- **Изисква:** Group By
- **Пример:** Производителност в различни метрики

**Radial Bar**
- **Използвайте за:** Кръгли индикатори за напредък показващи проценти (0-100%)
- **Не използва:** Group By или Group By Period
- **Изчисление на проценти:** Добавете условия, след това активирайте "Изчисли като процент" за автоматично изчисляване на (изпълнени условия / общо) × 100
- **Пример:** Коефициент на конверсия на лийдове (15%), успех при доставка на SMS (82.8%), завършване на цели

## Table Widget-и

Показване на подробни записи с персонализирани колони.

### Стъпка 1: Основни настройки

**Таблица с данни**
- Изберете източник: calls, leads, campaigns, assistants, phone_numbers, sms

**Филтър за кампания** (само за leads таблица)
- Изберете кампания за да:
  - Филтрирате таблицата към тази кампания
  - Активирате колони с персонализирани променливи
- Незадължително

**Филтър за асистент** (само за calls таблица)
- Изберете асистент за да:
  - Филтрирате таблицата към този асистент
  - Активирате колони с полета за оценка
- Незадължително

**Избор на колони**
- Основни колони: Id, Status, Created At, др.
- Колони за връзки: Assistant Name, Campaign Name
- Полета за оценка: evaluation.{field_name} (ако е избран асистент)
- Променливи на лийдове: variables.{variable_name} (ако е избрана кампания)
- По подразбиране ако няма избрани: Id, Status, Created At

### Стъпка 2: Показване и филтри

**Етикет** (задължителен)
- Заглавие на таблицата

**Описание**
- Незадължителен подзаглавен текст

**Интервал от дати**
- Същото като статистики/диаграми

**Сортиране по**
- Колона за сортиране по подразбиране
- Само прости колони (не връзки)
- По подразбиране: created_at

**Посока на сортиране**
- desc (най-новите първи) - по подразбиране
- asc (най-старите първи)

**Редове на страница**
- Брой редове за показване (1-100)
- По подразбиране: 10

**Условия**
- Същото като статистики/диаграми

### Интерактивни функции

**Кликаеми редове**
- Редовете на таблицата са кликаеми
- Автоматично препраща към страницата за детайли/редактиране на записа
- Работи за: Повиквания, Лийдове, Кампании, Асистенти

**Живо филтриране по дата**
- Променяйте интервала от дати директно на widget-а
- Заменя конфигурирания интервал по подразбиране
- Полезно за изследване на различни периоди от време

**Филтър по статус**
- Падащо меню с множество избор за филтриране по статус
- Налично за: Повиквания, Лийдове, Кампании, Асистенти, SMS
- Изберете множество статуси наведнъж

**Показване на текст**
- Дълъг текст се съкращава автоматично на 50 знака
- Посочването върху съкратен текст показва пълното съдържание в подсказка

### Типове колони

**Колони за телефонни номера**
- Международен формат (+1 555 123 4567)
- Префикс със знаме на държавата
- Примери: client_phone_number, phone_number

**Колони за статус**
- Цветни значки
- Примери: поле status в която и да е таблица

**Колони за дати**
- Формат "преди 3 часа"
- Посочването показва пълната дата и час
- Примери: created_at, updated_at

**Колони за оценка** (calls таблица с асистент)
- Boolean полета: отметка или X със зелена/червена значка
- Не-boolean полета: стойност със сива значка
- Посочването показва описанието на полето
- Формат: evaluation.{field_name}

**Колони за променливи** (leads таблица с кампания)
- Персонализирани данни за лийдове
- Формат: variables.{variable_name}
- Примери: customer_name, company

**Колони за връзки**
- Достъп до свързани данни
- Примери: Assistant Name, Campaign Name
- Не може да се сортира по тях

### Примери

**Последни повиквания**
```
Таблица: calls
Колони:
  - Client Phone Number
  - Assistant Name
  - Duration
  - Status
  - Created At
Сортиране: Created At desc
Редове: 25
Интервал от дати: 7 дни
```

**Успешни срещи**

Асистент: [Изберете за зареждане на полета]
Колони:

Условия:

    Variable Value: true

## Филтриране на данни

Всички widget-и поддържат условия за филтриране на данните.

### Оператори

- `=` - Равно
- `!=` - Не е равно
- `>` - По-голямо от
- `>=` - По-голямо или равно
- `<` - По-малко от
- `<=` - По-малко или равно
- `like` - Съдържа текст
- `in` - Едно от (разделени със запетая)
- `not in` - Не едно от
- `is null` - Няма стойност
- `is not null` - Има стойност

### Основно филтриране

**Филтър за статус**
```
Колона: Status
Оператор: =
Стойност: completed
```

**Филтър за продължителност**
```
Колона: Duration
Оператор: >
Стойност: 60
```

**Множество стойности**
```
Колона: Status
Оператор: in
Стойност: completed,in-progress
```

### Филтриране по полета за оценка

Филтрирайте по стойности от оценка след повикване.

**Настройка:**
1. Колона: Extracted Variables
2. Extracted Variable Name: Status или Custom variable
3. Ако Custom variable: Въведете името на полето в Custom Variable Name
4. Variable Value: true/false за boolean, или any + персонализирана стойност

**Boolean пример - Резервирана среща**
```
Колона: Extracted Variables
Extracted Variable Name: Custom variable
Custom Variable Name: appointment_booked
Оператор: =
Variable Value: true
```

**Не-boolean пример - Настроение**
```
Колона: Extracted Variables
Extracted Variable Name: Custom variable
Custom Variable Name: sentiment
Оператор: =
Variable Value: any
Custom Value: positive
```

**Проверка дали полето съществува**
```
Колона: Extracted Variables
Extracted Variable Name: Custom variable
Custom Variable Name: interested
Оператор: =
Variable Value: any
```

### Множество условия

Условията се комбинират с AND:

```
Филтър за успешни дълги повиквания:
Условие 1:
  - Колона: Status
  - Оператор: =
  - Стойност: completed

Условие 2:
  - Колона: Duration
  - Оператор: >
  - Стойност: 120

Условие 3:
  - Колона: Extracted Variables
  - Extracted Variable Name: Custom variable
  - Custom Variable Name: appointment_booked
  - Оператор: =
  - Variable Value: true
```
