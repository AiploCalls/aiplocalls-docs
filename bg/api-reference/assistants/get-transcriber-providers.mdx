api: "GET /user/assistants/transcriber-providers"

---

Този endpoint връща списък с наличните доставчици на транскриптор, които могат да бъдат използвани при създаване или обновяване на асистенти. Използвайте върнатия `id` като `transcriber_provider_id` при конфигуриране на асистент.

### Параметри за заявка

<ParamField query="language_id" type="integer">
  Филтрира доставчиците по поддръжка на език. Връща само доставчици, които поддържат дадения език. Използвайте [Get Languages](/api-reference/assistants/get-languages) endpoint, за да получите наличните ID на езици.
</ParamField>

### Полета в отговора

<ResponseField name="data" type="array">
  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      Уникалният идентификатор на доставчика. Използвайте това като `transcriber_provider_id` при създаване или обновяване на асистенти.
    </ResponseField>

    <ResponseField name="name" type="string">
      Показваното име на доставчика (например "Deepgram", "Azure", "Gladia")
    </ResponseField>

    <ResponseField name="code" type="string">
      Вътрешният код на доставчика
    </ResponseField>
  </Expandable>
</ResponseField>

### Забележки

- Ако не е предоставен филтър `language_id`, се връщат всички доставчици
- Не всички доставчици поддържат всички езици — използвайте филтъра `language_id`, за да получите съвместими доставчици
- Доставчиците на транскриптор са релевантни само за `pipeline` режим (multimodal и dualplex режимите обработват STT вътрешно)
- Ако не зададете `transcriber_provider_id` при създаване на асистент, се използва стандартният доставчик за езика
