---

Този endpoint връща списък с наличните AI модели, които могат да се използват при създаване или актуализиране на асистенти.

### Query Parameters

<ParamField query="type" type="string" default="llm">
  Типът модели за извличане въз основа на режима на работа:
  - `llm` - LLM модели за **pipeline** режим (по подразбиране)
  - `multimodal` - Multimodal модели за **multimodal** режим
  - `dualplex` - Multimodal модели за **dualplex** режим
</ParamField>

### Полета в отговора

<ResponseField name="data" type="array">
  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      Уникалният идентификатор на модела
    </ResponseField>

    <ResponseField name="name" type="string">
      Показваното име на модела
    </ResponseField>

    <ResponseField name="code" type="string">
      Вътрешният код (само за multimodal/dualplex модели)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>

```json LLM Models (type=llm)
[

    "name": "GPT-4o-mini"
```

### Забележки

- Ако не е предоставен параметър `type`, по подразбиране се връщат LLM модели
- Използвайте полето `llm_model_id` при създаване на **pipeline** асистенти
- Използвайте полето `multimodal_model_id` при създаване на **multimodal** или **dualplex** асистенти
