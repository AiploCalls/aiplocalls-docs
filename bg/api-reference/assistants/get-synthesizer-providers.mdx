description: "Извличане на достъпни доставчици на TTS (text-to-speech) синтезатори"
---

Този endpoint връща списък с достъпни доставчици на синтезатори, които могат да бъдат използвани при създаване или актуализиране на асистенти. Използвайте връщаният `id` като `synthesizer_provider_id` при конфигуриране на асистент.

### Query параметри

<ParamField query="language_id" type="integer">
  Филтриране на доставчиците по поддръжка на езици. Връща само доставчиците, които поддържат зададения език. Използвайте [Get Languages](/api-reference/assistants/get-languages) endpoint-а за получаване на достъпни ID на езици.
</ParamField>

### Response полета

<ResponseField name="data" type="array">
  <Expandable title="properties">
    <ResponseField name="id" type="integer">
      Уникалният идентификатор на доставчика. Използвайте го като `synthesizer_provider_id` при създаване или актуализиране на асистенти.
    </ResponseField>

    <ResponseField name="name" type="string">
      Показваното име на доставчика (например "ElevenLabs", "Azure", "Cartesia")
    </ResponseField>

    <ResponseField name="code" type="string">
      Вътрешният код на доставчика
    </ResponseField>
  </Expandable>
</ResponseField>

### Забележки

- Ако не е предоставен филтър `language_id`, се връщат всички доставчици
- Не всички доставчици поддържат всички езици — използвайте филтъра `language_id` за получаване на съвместими доставчици
- Доставчиците на синтезатори са актуални само за режимите `pipeline` и `dualplex` (multimodal режимът обработва TTS вътрешно)
- Ако не зададете `synthesizer_provider_id` при създаване на асистент, се използва доставчикът по подразбиране за езика
