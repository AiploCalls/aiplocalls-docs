Решения за често срещани проблеми с интеграцията на Cal.com.

## Проблеми със Свързването

### Невалиден API Key

**Проблем:** Събитията не се зареждат или синхронизацията се проваля

**Решение:**
- Проверете дали API key-ят е активен в Cal.com → Settings → Developer → API Keys
- Уверете се, че ключът включва префикса `cal_live_`
- Генерирайте нов API key ако е необходимо

### Събитията Не Се Зареждат

**Проблем:** Падащото меню със събития е празно след въвеждане на API key

**Решение:**
- Проверете дали API key-ят е правилен
- Кликнете извън полето с API key за да се задейства зареждането на събития
- Опреснете страницата ако събитията все още не се появят
- Проверете дали имате създадени събития в Cal.com

## Проблеми със Синхронизацията

### Синхронизирането на Събитие Се Провали

**Проблем:** Бутонът "Sync Event" се проваля или показва грешка

**Решение:**
1. Кликнете върху бутона **"Troubleshoot"** вместо това
2. Това нулира всички полета за резервация до правилната конфигурация
3. Проверете промените в настройките на събитието в Cal.com

### Грешки в Конфигурацията на Полетата

**Проблем:** Получавате грешки относно задължителни полета или липсващи полета

**Решение:**
- Кликнете **"Troubleshoot"** за да нулирате полетата
- Само име, имейл, телефон трябва да са задължителни
- Всички други полета (местоположение, гости и т.н.) трябва да са по избор

## Провали при Резервации

### Не Се Изпращат Календарни Покани

**Проблем:** Резервациите успяват, но клиентът не получава покана

**Решение:**
- Дефинирайте променливата `email` в [настройките на асистента](/ai-assistants/settings/prompt-and-tools#call-variables)
- Подавайте имейла на клиента при създаване на leads
- Проверете дали форматът на имейла е валиден

### Липсва Телефонен Номер

**Проблем:** Телефонният номер не се появява в календарните събития

**Решение:**
- Кликнете върху бутона **"Sync Event"** или **"Troubleshoot"**
- Това автоматично добавя полето за телефон към събитието
- Телефонният номер е включен след синхронизацията

### Проблеми с Часовата Зона при Резервации

**Проблем:** Срещите се планират в грешно време

**Решение:**
- Проверете часовата зона в настройките на събитието в Cal.com
- Проверете дали асистентът получава правилното време от клиента
- Тествайте с конкретни споменавания на време по време на разговора

## Проблеми с Тестването

### Резервациите с Web Call Се Провалят

**Проблем:** Резервациите работят непоследователно с web calls

**Решение:**
- Използвайте истински телефонни обаждания за тестване (не web calls)
- Web calls може да не подават променливата за телефона правилно
- При използване на web calls, подавайте телефона в международен формат с + префикс (напр. "+1 555-123-4567")
- Телефонните обаждания осигуряват правилното предаване на всички променливи

### Множество Календари Не Работят

**Проблем:** Само един календар работи или събитията конфликтуват

**Решение:**
- Всеки календар трябва да има избрано различно събитие
- Използвайте различни имена за събитията (напр. "15min", "30min")
- Синхронизирайте всяко събитие отделно след добавяне

## Получаване на Помощ

Ако проблемите продължават:

1. Проверете транскриптите на обажданията за опити за резервация
2. Проверете дали събитието от Cal.com се показва в dashboard-а на Cal.com
3. Тествайте с просто телефонно обаждане за резервация
4. Свържете се с поддръжката с:
   - Снимка на екрана с грешката
   - Снимка на екрана с настройките на събитието в Cal.com
   - Транскрипт на обаждането показващ опита за резервация
