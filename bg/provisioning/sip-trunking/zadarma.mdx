Това ръководство ви води през свързването на вашите Zadarma телефонни номера използвайки SIP trunking. Zadarma предлага два типа връзки — **PBX** и **SIP Direct** — и настройката се различава леко за всеки.

<Tip>
**Кой тип имате?** Ако използвате виртуалната PBX на Zadarma с вътрешни номера (напр., 100, 101), имате **PBX**. Ако използвате самостоятелен SIP акаунт без PBX, имате **SIP Direct**.
</Tip>

Изберете вашия тип връзка по-долу и следвайте стъпките:

<Tabs>
  <Tab title="PBX">
    ## Стъпка 1: Съберете вашите SIP данни за вход

    1. Влезте в вашия [Zadarma Dashboard](https://my.zadarma.com)
    2. Отидете на **My PBX** → **Extensions**

    <Frame>
      <img
        src="/resources/zadarma-mypbx-extensions.png"
        alt="Zadarma My PBX Extensions"
      />
    </Frame>

    3. Намерете вашия вътрешен номер и отбележете тези детайли:

    ```
    Extension: напр., 510000-100
    Server address: pbx.zadarma.com
    Username: Вашето потребителско име за вътрешен номер (напр., 510000-100)
    Password: Вашата парола за вътрешен номер
    Phone number: Вашият телефонен номер (напр., +521151900966)
    ```

    <Frame>
      <img
        src="/resources/zadarma-switch-extension-settings.png"
        alt="Zadarma PBX Extension Settings"
      />
    </Frame>

    ## Стъпка 2: Настройте External Server (за входящи обаждания)

    За да получавате входящи обаждания, трябва да конфигурирате **External Server** в Zadarma, който препраща входящи обаждания към SIP сървъра на платформата.

    1. В вашия [Zadarma Dashboard](https://my.zadarma.com), отидете на **Settings** → [**Virtual phone numbers**](https://my.zadarma.com/dirnum/active/)

    <Frame>
      <img
        src="/resources/zadarma-settings-virtual-numbers.png"
        alt="Zadarma Settings Virtual phone numbers"
      />
    </Frame>

    2. Намерете номера, който искате да свържете и кликнете върху **иконата със зъбчато колело**, за да отворите неговите настройки

    <Frame>
      <img
        src="/resources/zadarma-settings-virtual-numbers-gear-icon.png"
        alt="Zadarma Settings Virtual phone numbers Gear Icon"
      />
    </Frame>

    3. Кликнете **External Server** и активирайте опцията **SIP URI**

    <Frame>
      <img
        src="/resources/zadarma-pbx-external-server.png"
        alt="Zadarma PBX External Server SIP URI configuration"
      />
    </Frame>

    4. Въведете адреса на SIP сървъра използвайки вашия **телефонен номер с префикс `+`**:

    ```
    +YOUR_PHONE_NUMBER@5wed79aeavr.sip.livekit.cloud
    ```

    Например, ако вашият номер е `+551151900966`:

    ```
    +521151900966@5wed79aeavr.sip.livekit.cloud
    ```

    <Frame>
      <img
        src="/resources/zadarma-pbx-server-address.png"
        alt="Zadarma PBX Server Address configuration"
      />
    </Frame>

    5. Кликнете **Show IP** под адреса на сървъра, за да разкриете IP адресите на Zadarma (напр., `185.45.152.216`, `185.45.155.33`)

    <Frame>
      <img
        src="/resources/zadarma-pbx-show-ip-button.png"
        alt="Zadarma PBX Show IP button"
      />
    </Frame>

    <Frame>
      <img
        src="/resources/zadarma-pbx-show-ip-addresses.png"
        alt="Zadarma PBX IP addresses revealed"
      />
    </Frame>

    <Note>
    **Запазете тези IP адреси** — ще ви трябват в Стъпка 3 при конфигурирането на входяща оторизация в платформата.
    </Note>

    <Warning>
    **PBX потребителите трябва да използват телефонния номер с `+`** в адреса на External Server. Използването на номера на PBX вътрешната линия (напр., `533177-100`) **няма** да работи за входящи обаждания.
    </Warning>

    ## Стъпка 3: Импортирайте номерата в платформата

    Сега импортирайте номера, за да знае платформата как да насочва обажданията.

    1. Отидете директно на страницата с телефонни номера в таблото си:

       **[https://call.aiployees.com/phone-numbers](https://call.aiployees.com/phone-numbers)**

       Ако ви отведе до страницата за вход, първо влезте — автоматично ще ви пренасочи към страницата с телефонни номера.

    2. Кликнете **"Integrate SIP trunk"** (горен десен ъгъл на страницата)

    <Frame>
      <img
        src="/resources/zadarma-import-sip.png"
        alt="Import numbers via SIP trunk"
      />
    </Frame>

    3. Попълнете вашия телефонен номер и SIP данни за вход:

    ```
    Phone number: напр., +551151900966
    Username: Вашият PBX вътрешен номер (напр., 533100-100)
    Your SIP Extension: Вашият телефонен номер (напр., +551151900966)
    Password: Вашата парола за PBX вътрешен номер
    ```

    <Warning>
    **PBX потребители:** **Username** е вашият PBX вътрешен номер (напр., `533100-100`). Полето **"Your SIP Extension"** трябва да бъде вашият **телефонен номер** — това трябва да съвпада с номера, който използвахте в адреса на External Server в Стъпка 2.
    </Warning>

    <Frame>
      <img
        src="/resources/zadarma-pbx-auth.png"
        alt="PBX Extension credentials"
      />
    </Frame>

    4. Въведете **Server address** за изходящи обаждания:

    ```
    pbx.zadarma.com
    ```

    <Frame>
      <img
        src="/resources/zadarma-pbx-outbound-settings.png"
        alt="Zadarma outbound server address"
      />
    </Frame>

    5. Изберете **типа оторизация**. Изберете **IP** и добавете IP адресите на Zadarma, които отбелязахте по-рано от **Show IP** в Стъпка 2.

    <Frame>
      <img
        src="/resources/zadarma-ip-inbound.png"
        alt="Zadarma IP Authorization"
      />
    </Frame>

    6. Изберете **държавата**, в която се намира вашият SIP trunk.

    <Frame>
      <img
        src="/resources/zadarma-sip-country.png"
        alt="SIP Country selection"
      />
    </Frame>

## Изисквания за сигурност

Вашите SIP данни за вход трябва да бъдат сигурни:

- **Потребителско име**: Номерът на вашата вътрешна линия или потребителско име, предоставено от доставчика
- **Паролата трябва да има**:
  - 12 или повече символа
  - Цифри
  - Главни и малки букви
  - Препоръчват се специални символи

## Ограничения при трансфер на обаждания

<Warning>
**Zadarma не поддържа SIP REFER**, което означава, че стандартният студен трансфер няма да работи директно. Топлият трансфер работи автоматично — платформата го обработва чрез SIP INVITE.

За **студен трансфер** можете да използвате опцията **Custom SIP Transfer** в настройките за трансфер на вашия асистент. Задайте персонализирания SIP URI на:

```
sip:+DESTINATION_NUMBER@sip.zadarma.com
```

Например: `sip:+15557654321@sip.zadarma.com`

Уверете се, че номерът е в международен E.164 формат с префикс `+`. **Не** използвайте `tel:` URI формат.
</Warning>

## Често срещани проблеми

<AccordionGroup>
  <Accordion title="Входящите обаждания не работят">
    - **Проверете адреса на вашия External Server** — уверете се, че форматът съответства на вашия тип връзка:
      - **PBX**: `+YOUR_PHONE_NUMBER@5wed79aeavr.sip.livekit.cloud` (телефонен номер с `+`)
      - **SIP Direct**: `YOUR_EXTENSION@5wed79aeavr.sip.livekit.cloud` (номер на вътрешна линия)
    - **Проверете дали External Server е активиран** в настройките на Zadarma
    - **Проверете дали полето "Your SIP Extension" в платформата съвпада**: телефонен номер за PBX, вътрешна линия за SIP Direct
  </Accordion>

  <Accordion title="Изходящите обаждания не работят">
    - **Проверете Server address** — трябва да бъде `pbx.zadarma.com` за PBX или `sip.zadarma.com` за SIP Direct
    - **Проверете вашите данни за вход** — уверете се, че потребителското име и паролата съвпадат с това, което е показано в SIP настройките на Zadarma
    - **Проверете баланса си в Zadarma** — изходящите обаждания изискват достатъчен кредит
  </Accordion>

  <Accordion title="Студеният трансфер не работи">
    Zadarma не поддържа SIP REFER. За да активирате студен трансфер:
    1. Отидете в настройките на инструмента **Transfer Phone Call** на вашия асистент
    2. Активирайте **"Custom SIP transfer"** (под Advanced)
    3. Задайте SIP URI на `sip:+DESTINATION_NUMBER@sip.zadarma.com`

    Ако студеният трансфер все още не работи, проверете дали номерът на местоназначението е в международен формат с `+`.
  </Accordion>

  <Accordion title="Кой тип връзка трябва да използвам — PBX или SIP Direct?">
    - **PBX** се препоръчва, ако имате нужда от множество вътрешни линии, правила за насочване на обаждания, IVR менюта или други PBX функции в Zadarma
    - **SIP Direct** е по-простo, ако просто имате нужда от един номер, свързан без PBX функции

    И двата работят еднакво добре за входящи и изходящи AI обаждания.
  </Accordion>
</AccordionGroup>
